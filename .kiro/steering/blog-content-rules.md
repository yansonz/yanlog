# Blog Content Rules

## Tag Guidelines
- Tags must always be in lowercase
- Use hyphens for multi-word tags (e.g., "web-development", "machine-learning")
- Keep tags concise and relevant
- Avoid special characters except hyphens

## Examples
✅ Good:
```yaml
tags: ["kiro", "developer", "community", "character"]
tags: ["aws", "cloud-computing", "infrastructure"]
```

❌ Bad:
```yaml
tags: ["Kiro", "Developer", "Community", "Character"]
tags: ["AWS", "Cloud Computing", "Infrastructure"]
```

## Content Structure
- Use consistent heading hierarchy (H2 for main sections, H3 for subsections)
- Include frontmatter with all required fields
- Maintain consistent spacing and formatting
- Use proper Korean spacing rules for Korean content

## Image Guidelines
- Use WebP format for optimal performance
- Host images on the image server (images.yanbert.com)
- Include descriptive alt text
- Specify width/height attributes when needed
