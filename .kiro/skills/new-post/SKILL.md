# New Post Creation Skill

새 블로그 포스트를 생성하는 워크플로우입니다.

## Slug 규칙

기존 포스트 파일명을 기준으로 slug를 만듭니다.

**형식:** `{주제-키워드}-{YYYYMMDD}`

**예시:**
- `kiro-user-interview-003-20260608`
- `aws-london-summit-2026-20260422`
- `crossfit-open-26-1-in-portugal-20260315`

**규칙:**
- 모두 소문자
- 단어 구분은 하이픈(`-`)
- 끝에 날짜 8자리 (오늘 날짜 기준)
- 한글 금지, 영어 키워드만 사용
- 시리즈물은 번호 포함 (예: `kiro-user-interview-003-20260315`)

## 워크플로우

### 0단계: 포스트 정보 확인

스킬이 호출되면 가장 먼저 아래 정보가 충분한지 확인한다.
정보가 없거나 부족하면 **반드시 사용자에게 먼저 물어본다.**

```
확인할 항목:
1. 어떤 주제/내용의 글인가?
2. 포스트 날짜는 언제인가? (없으면 오늘 날짜 사용)
3. 시리즈물인가? (예: 인터뷰 시리즈, 이벤트 시리즈 등)
```

예시 질문:
> "어떤 글을 작성할 예정인가요? 주제와 날짜를 알려주시면 slug를 제안해 드릴게요."

정보가 충분하면 다음 단계로 진행한다.

### 1단계: 한글 포스트 생성

사용자가 새 포스트 작성을 요청하면:

1. 주제를 파악해서 slug 후보를 제안한다
2. 사용자 확인 후 아래 명령어 실행:

```bash
node scripts/new-post.js {slug} ko
```

3. 생성된 파일 경로를 안내한다:
   - `content/posts/ko/{slug}.mdx`
4. `draft: true` 상태로 생성됨을 알린다
5. **여기서 멈춘다** — 이후 사람이 직접 내용을 작성한다

### 2단계: 교정 (사람 + AI 협업)

한글 포스트 작성이 완료되면 사용자가 교정을 요청한다.
교정 항목:
- 맞춤법 및 띄어쓰기
- 문장 자연스러움
- 팩트체크 (필요 시 웹 검색)
- 톤앤매너 일관성

교정은 여러 차례 반복될 수 있다. 사용자가 완성됐다고 할 때까지 계속한다.

### 3단계: 영문 포스트 생성

사용자가 "영문 포스트 생성"을 요청하면:

1. 한글 포스트를 읽는다
2. `.kiro/steering/english-translation.md` 규칙을 따른다
3. 아래 명령어로 빈 파일 먼저 생성:

```bash
node scripts/new-post.js {slug} en
```

4. 영문 번역본을 작성해서 파일에 저장한다
5. frontmatter의 `locale`을 `"en"`으로, `title`과 `description`도 영문으로 작성한다

## 주의사항

- 영문 포스트는 한글 포스트가 완성된 이후에만 생성한다
- slug는 한글/영문 포스트가 동일하게 사용한다
- `draft: true`는 사용자가 직접 `false`로 변경하거나 제거한다
- 이미지가 있으면 `upload-image-to-server` 스킬을 참고한다
